# 记忆库备份：RK3588 核心板配置与网络连接 (RK3588 Board and Connection)

## 1. 硬件与系统环境
- **硬件平台**：深圳易百纳 ELF2 RK3588 开发板。
- **系统版本**：`Ubuntu 22.04.5 LTS (GNU/Linux 5.10.209 aarch64)`。
- **ROS 2 运行版本**：`ROS 2 Humble Hawksbill` (使用主机 APT 源及 Docker 部署)。
- **存储方案**：主要工作空间及数据挂载于 `/mnt/sdcard/` 目录下（大容量 SD 卡/外置存储），包括：
  - `/mnt/sdcard/medicine_robot_ws`：核心 ROS2 送药小车工作空间。
  - `/mnt/sdcard/medicine_robot_data`：保存地图、YOLO 模型和运维日志的文件夹。

## 2. 局域网网络连接
- **连接方式**：Wi-Fi 连接（物理网卡 `wlan0`）。
- **默认连接 SSID**：`Xiaomi_AD81`。
- **板端局域网固定/当前分配 IP**：`192.168.31.125`。
- **SSH 访问权限**：
  - **SSH 用户名**：`elf` （密码：`elf`）。
  - **SSH 端口**：`22`。
  - *注意*：`root` 账号在 SSH 默认配置下禁止直接登录，请通过普通用户 `elf` 登录后执行 `sudo` 或切换到 `root` 权限。

## 3. 开机自启与进程守护 (8085 业务服务)
- 板载已部署守护进程：
  - 正式业务服务运行于默认 `ROS_DOMAIN_ID = 0`，占用局域网 Web 端口 `8085`。
  - 若在不接入真实下位机硬件的情况下启动，系统业务状态（`api/state`）默认为 `simulated`（仿真模式），完全不阻塞前端页面流程及上层语音、视觉业务的联调。
  - 测试期间如果需要使用独立的 ROS 隔离环境，通常使用独立的 `ROS_DOMAIN_ID`（如 `91` 或 `93`）配合非标准端口（如 `8095`），防止干扰正常运行的 8085 正式配送系统。
  - 临时测试完毕后，需通过 PID 进程管理安全退出，确保没有残留后台进程占用串口（`/dev/ttyS9`）。
