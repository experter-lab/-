# Cartographer localization notes

- Nav2 static map must use the map exported from the same pbstream that Cartographer localization loads.
- Current static map path: `/mnt/sdcard/medicine_robot_data/maps/rk3588_carto_latest_static.yaml`.
- Current pbstream path: `/mnt/sdcard/medicine_robot_data/maps/rk3588_carto_latest.pbstream`.
- `rk3588_start_nav2_carto.sh` no longer starts `carto_relocalize_bridge.py` by default. The bridge timed out/crashed on the RK3588 and left `/initialpose` unwired, so keep `ENABLE_RELOC_BRIDGE=0` until a one-shot relocalizer replaces it.
- RViz `2D Pose Estimate` publishes `/initialpose`, but it has no effect unless a relocalization subscriber is intentionally started.
- The bridge must not depend entirely on `/get_trajectory_states`; on this RK3588 it can timeout even while Cartographer is alive. The script now falls back to the tracked active trajectory ID, normally starting from 1.
- Do not wait synchronously for Cartographer services inside the `/initialpose` callback. The bridge now runs relocalization in a background thread and polls futures while the main executor keeps spinning.
- RViz TF axes can look like several robot poses. `rk3588_nav2_view.rviz` now disables the TF display by default; turn it on only when debugging frame links.
- RViz `TopDownOrtho` must use `Target Frame: map`. If it targets `base_link`, the camera follows Cartographer pose jitter and the static map appears to shake even when `/map_static` itself is stable.
- During localization debugging, keep `nav2_rviz_plugins/GoalTool` out of `rk3588_nav2_view.rviz`. The user accidentally triggered `Nav2 Goal` while trying to use `2D Pose Estimate`; bridge logs showed no `/initialpose` message, while RViz logged `Start navigation`.
- Cartographer stable localization profile on 2026-06-09 reduced stationary TF jitter from roughly x range 8.7 cm / yaw range 3.3 deg to roughly x-y range 2 cm / yaw range 1 deg.
- After local scan-to-map scoring on 2026-06-09, current alignment improved from about 9 cm residual to about 3 cm residual after applying a refined initial pose. This is acceptable for first Nav2 checks, but not yet proof that the map is clean.
- If Cartographer logs `0 additional constraints` and `map -> base_link` drifts while the robot is stationary, it is not locked to the frozen map yet. Do not enable Nav2 drive until scan overlays the map after an initial pose reset.
- Aggressive global search (`POSE_GRAPH.global_sampling_ratio = 1.0` with a wide search window) can make Cartographer hang on the RK3588. Use a moderate relocalization profile and manual initial pose first.
