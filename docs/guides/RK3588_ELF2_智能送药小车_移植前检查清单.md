### 2.3 Jazzy -> Humble 兼容性预检

当前开发环境和目标板环境不同：

```text
PC/WSL2：Ubuntu 24.04 + ROS 2 Jazzy + Python 3.12 + x86_64
RK3588：Ubuntu 22.04 + ROS 2 Humble + Python 3.10 + ARM64
```

主要风险：

```text
[ ] ROS 2 Jazzy 与 Humble 的 launch/API 差异
[ ] Python 3.12 到 Python 3.10 的语法/标准库差异
[ ] x86_64 到 ARM64 的 Python wheel/原生库差异
[ ] zxingcpp、OCR、AIKit 等 native 依赖在 ARM64 上不可直接复用
[ ] msg/srv 修改后旧生成接口残留
```

当前代码准备原则：

```text
1. 以后新增代码按 Humble + Python 3.10 作为最低兼容目标
2. launch 文件统一使用 executable/name，不使用旧字段 node_executable/node_name
3. 修改 *.msg/*.srv 后必须清理 medicine_interfaces 和依赖包重新 build
4. RK3588 首轮先验证任务与 Dashboard，不同时打开摄像头、OCR、语音、导航
5. zxingcpp 在 ARM64 上单独验证，不能阻塞主闭环
```

## 9. 第二阶段：API 验证患者清单闭环

在 RK3588 上执行：

```bash
python3 - <<'PY'
import json, time, urllib.request
base='http://127.0.0.1:8080'

def get(path):
    with urllib.request.urlopen(base+path, timeout=5) as r:
        return json.loads(r.read().decode())

def post(path, payload):
    data=json.dumps(payload, ensure_ascii=False).encode()
    req=urllib.request.Request(base+path, data=data, headers={'Content-Type':'application/json'}, method='POST')
    with urllib.request.urlopen(req, timeout=8) as r:
        return json.loads(r.read().decode())

orders=get('/api/patient_orders')['orders']
order=orders[0]
medications=order['medications']
payload={
    'target_station': order['target_station'],
    'source_station': 'pharmacy',
    'medicine_name': '、'.join(m['medicine_name'] for m in medications),
    'patient_id': order['patient_id'],
    'patient_name': order['patient_name'],
    'ward_id': order['ward_id'],
    'bed_no': order['bed_no'],
    'product_code': medications[0]['product_code'],
    'product_model': medications[0]['product_model'],
    'quantity': medications[0]['quantity'],
    'trace_id': medications[0]['trace_id'],
    'order_no': medications[0]['order_no'],
    'medications_json': json.dumps(medications, ensure_ascii=False),
}
created=post('/api/tasks', payload)
print('created', created)
task_id=created.get('task_id','')
print('load1', post('/api/verify_task', {'task_id': task_id, 'product_code': '43043', 'trace_id': '', 'stage': 'load'}))
print('load2', post('/api/verify_task', {'task_id': task_id, 'product_code': 'C177248', 'trace_id': '202011444', 'stage': 'load'}))
time.sleep(4)
print('dispense1', post('/api/verify_task', {'task_id': task_id, 'product_code': '43043', 'trace_id': '', 'stage': 'dispense'}))
print('dispense2', post('/api/verify_task', {'task_id': task_id, 'product_code': 'C177248', 'trace_id': '202011444', 'stage': 'dispense'}))
time.sleep(1)
state=get('/api/state')
print('final', {k: state.get(k) for k in ['state','message','patient_id','patient_name','bed_no','medication_total_count','medication_loaded_count','medication_dispensed_count']})
PY
```

[ ] 到站后进入 WAITING_DISPENSE_CONFIRMATION
